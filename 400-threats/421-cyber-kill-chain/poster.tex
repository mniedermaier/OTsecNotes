\section{\textcolor{accent}{\faIcon{info-circle}}\hspace{0.4em}Overview}

The Cyber Kill Chain framework maps attack stages to help identify where to detect and disrupt attacks. OT attacks require \textbf{more stages than IT attacks} because adversaries must understand physical processes before impact. This extended timeline provides more \textcolor{success}{\faIcon{eye}}\hspace{0.2em}detection opportunities.

\section{\textcolor{accent}{\faIcon{sitemap}}\hspace{0.4em}SANS ICS Kill Chain}

\posterinfo{
The SANS ICS Kill Chain recognizes that attacks on industrial systems require \textbf{two distinct phases}: initial IT intrusion followed by ICS-specific attack development and execution.
}

\subsection{\textcolor{accent}{\faIcon{desktop}}\hspace{0.3em}Stage 1: IT/Corporate Intrusion}

Follows traditional kill chain to gain corporate access:

\begin{itemize}
    \item \textcolor{info}{\faIcon{desktop}}\hspace{0.2em}Target IT systems, email, corporate network
    \item \textcolor{info}{\faIcon{crosshairs}}\hspace{0.2em}Goal: Establish foothold and pivot toward OT
    \item \textcolor{info}{\faIcon{clock}}\hspace{0.2em}May take weeks to months of patient operation
\end{itemize}

}

\posterwarning{
\textbf{Defense in depth:} No single control stops all attacks. Implement detection and prevention at multiple stages. Each disrupted stage forces the attacker to restart or adapt.
}

\section{\textcolor{accent}{\faIcon{crosshairs}}\hspace{0.4em}MITRE ATT\&CK for ICS}

Key ICS-specific tactics (beyond traditional IT):

\begin{itemize}
    \item \textcolor{info}{\faIcon{search}}\hspace{0.2em}\textbf{Discovery} -- Learning OT environment details
    \item \textcolor{warning}{\faIcon{arrows-alt-h}}\hspace{0.2em}\textbf{Lateral Movement} -- Moving through OT network
    \item \textcolor{info}{\faIcon{database}}\hspace{0.2em}\textbf{Collection} -- Gathering process data
    \item \textcolor{warning}{\faIcon{ban}}\hspace{0.2em}\textbf{Inhibit Response} -- Preventing operator action
    \item \textcolor{danger}{\faIcon{cogs}}\hspace{0.2em}\textbf{Impair Process} -- Disrupting physical operations
    \item \textcolor{danger}{\faIcon{exclamation-triangle}}\hspace{0.2em}\textbf{Impact} -- Causing damage or safety incidents
\end{itemize}

\posterdanger{
\textbf{Common gap:} Many organizations have IT monitoring but lack OT network visibility. Without OT-specific detection (protocol monitoring, behavioral analytics), Stage 2 of the ICS kill chain goes completely undetected.
}

\postertip{
OT attacks typically have an \textbf{extended timeline} -- months of reconnaissance before action. This is the defender's advantage. Detection during recon and lateral movement phases stops attacks before physical impact. Map your defenses to kill chain stages and identify gaps. Use MITRE ATT\&CK for ICS for detailed technique-level threat intelligence and detection rules.
}
