\section{\textcolor{accent}{\faIcon{info-circle}}\hspace{0.4em}Overview}

Supply chain attacks compromise \textbf{trusted vendors, software, hardware, or service providers} to reach targets indirectly. In OT, these attacks are particularly dangerous because industrial systems rely on specialized vendors with deep access, and security updates are applied infrequently.

OT supply chain risks are amplified by:

\begin{itemize}
    \item \textcolor{danger}{\faIcon{clock}}\hspace{0.2em}Long equipment lifecycles (15--25 years)
    \item \textcolor{danger}{\faIcon{key}}\hspace{0.2em}Vendor remote access for maintenance/support
    \item \textcolor{danger}{\faIcon{eye-slash}}\hspace{0.2em}Limited visibility into vendor security practices
    \item \textcolor{danger}{\faIcon{tools}}\hspace{0.2em}Delayed patching due to operational constraints
    \item \textcolor{danger}{\faIcon{link}}\hspace{0.2em}Dependence on specialized, sole-source vendors
\end{itemize}

\section{\textcolor{accent}{\faIcon{history}}\hspace{0.4em}Notable Incidents}

\subsection{\textcolor{accent}{\faIcon{bug}}\hspace{0.3em}Havex / Dragonfly (2013--2014)}

\begin{itemize}
    \item \textcolor{danger}{\faIcon{code}}\hspace{0.2em}Trojanized ICS vendor software installers
    \item \textcolor{danger}{\faIcon{globe}}\hspace{0.2em}Compromised legitimate vendor download sites
    \item \textcolor{danger}{\faIcon{search}}\hspace{0.2em}Havex malware scanned for OPC servers
    \item \textcolor{danger}{\faIcon{bolt}}\hspace{0.2em}Targeted energy sector organizations
\end{itemize}

\posterdanger{
Havex demonstrated that attackers could compromise ICS environments by \textbf{targeting vendors that asset owners trust implicitly}. Users downloading from official vendor websites received malware.
}

\begin{enumerate}
    \item \textcolor{danger}{\faIcon{microchip}}\hspace{0.2em}PLC/RTU firmware updates
    \item \textcolor{danger}{\faIcon{desktop}}\hspace{0.2em}HMI/SCADA software packages
    \item \textcolor{danger}{\faIcon{laptop}}\hspace{0.2em}Engineering workstation tools
    \item \textcolor{danger}{\faIcon{database}}\hspace{0.2em}Historian and OPC software
    \item \textcolor{danger}{\faIcon{network-wired}}\hspace{0.2em}Network equipment firmware
    \item \textcolor{danger}{\faIcon{key}}\hspace{0.2em}Vendor remote access tools
    \item \textcolor{danger}{\faIcon{hard-hat}}\hspace{0.2em}Safety system configurations
\end{enumerate}

\posterwarning{
\textbf{Vendor remote access} connections often bypass security controls and provide direct OT access. A compromised vendor uses legitimate credentials to access \textbf{multiple customer sites} simultaneously.
}

\section{\textcolor{accent}{\faIcon{shield-alt}}\hspace{0.4em}Defense Strategies}

\subsection{\textcolor{accent}{\faIcon{clipboard-check}}\hspace{0.3em}Vendor Risk Management}

\begin{itemize}
    \item \textcolor{info}{\faIcon{clipboard-check}}\hspace{0.2em}\textbf{Assessments} -- Evaluate vendor security before procurement
    \item \textcolor{info}{\faIcon{file-signature}}\hspace{0.2em}\textbf{Contracts} -- Include security obligations
    \item \textcolor{info}{\faIcon{chart-line}}\hspace{0.2em}\textbf{Monitoring} -- Track vendor security posture
    \item \textcolor{info}{\faIcon{bell}}\hspace{0.2em}\textbf{Notification} -- Require vendors to report incidents
\end{itemize}

\subsection{\textcolor{accent}{\faIcon{key}}\hspace{0.3em}Remote Access Controls}

\begin{itemize}
    \item \textcolor{success}{\faIcon{server}}\hspace{0.2em}Jump servers for all vendor access
    \item \textcolor{success}{\faIcon{fingerprint}}\hspace{0.2em}Multi-factor authentication required
    \item \textcolor{success}{\faIcon{video}}\hspace{0.2em}Session recording and monitoring
    \item \textcolor{success}{\faIcon{clock}}\hspace{0.2em}Time-limited access with approval
    \item \textcolor{success}{\faIcon{project-diagram}}\hspace{0.2em}Segment vendor from critical systems
\end{itemize}

\subsection{\textcolor{accent}{\faIcon{microchip}}\hspace{0.3em}Hardware Security}

\begin{itemize}
    \item \textcolor{success}{\faIcon{shopping-cart}}\hspace{0.2em}Purchase from authorized distributors only
    \item \textcolor{success}{\faIcon{search}}\hspace{0.2em}Inspect components for signs of tampering
    \item \textcolor{success}{\faIcon{check-circle}}\hspace{0.2em}Verify firmware integrity before deployment
    \item \textcolor{success}{\faIcon{list-alt}}\hspace{0.2em}Maintain inventory of hardware provenance
    \item \textcolor{success}{\faIcon{handshake}}\hspace{0.2em}Consider trusted supplier programs
\end{itemize}

\postersuccess{
\textbf{Zero trust for vendors:} Verify identity, limit access scope, monitor all activity, and assume any vendor could be compromised. Never allow persistent, unmonitored vendor connections to OT networks.
}

\postertip{
Supply chain attacks exploit trust relationships. Combine \textbf{vendor risk management, integrity verification, access controls, and segmentation} for defense in depth.
}
