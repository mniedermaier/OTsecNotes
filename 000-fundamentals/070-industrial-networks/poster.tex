\section{\textcolor{accent}{\faIcon{network-wired}}\hspace{0.4em}Overview}

Industrial networks connect field devices, controllers, and supervisory systems. Unlike IT networks optimized for throughput, industrial networks prioritize \textbf{determinism}, \textbf{reliability}, and \textbf{real-time performance} for process control.

\section{\textcolor{accent}{\faIcon{plug}}\hspace{0.4em}Serial Communications}

\begin{itemize}
    \item \textcolor{accent}{\faIcon{arrows-alt-h}}\hspace{0.2em}\textbf{RS-232:} Point-to-point, $<$15m, HMI-to-PLC, up to 115.2 kbps
    \item \textcolor{accent}{\faIcon{sitemap}}\hspace{0.2em}\textbf{RS-485:} Multi-drop (32 devices), up to 1200m, basis for Modbus RTU, up to 10 Mbps
\end{itemize}

\posterwarning{
Serial protocols have \textbf{no authentication or encryption}. Physical bus access allows reading and injecting traffic. Serial-to-Ethernet converters expose insecure protocols to IP networks.
}

\section{\textcolor{accent}{\faIcon{ethernet}}\hspace{0.4em}Industrial Ethernet}

Industrial Ethernet adapts standard Ethernet for factory and process automation with enhanced reliability and real-time capabilities.

\subsection{\textcolor{accent}{\faIcon{not-equal}}\hspace{0.3em}Key Differences from IT Ethernet}

\begin{itemize}
    \item \textcolor{accent}{\faIcon{clock}}\hspace{0.2em}\textbf{Deterministic timing:} Guaranteed delivery within defined time
    \item \textcolor{accent}{\faIcon{shield-alt}}\hspace{0.2em}\textbf{Ruggedized hardware:} Extended temp, vibration, EMI resistance
    \item \textcolor{accent}{\faIcon{copy}}\hspace{0.2em}\textbf{Ring redundancy:} Sub-50ms failover (MRP, HSR, PRP)
    \item \textcolor{accent}{\faIcon{layer-group}}\hspace{0.2em}\textbf{Application layer:} Industrial-specific protocols
\end{itemize}

\section{\textcolor{accent}{\faIcon{copy}}\hspace{0.4em}Network Redundancy}

\begin{center}

\begin{tabular}{p{1cm}p{3cm}p{2.5cm}}
\rowcolor{primary}
\textcolor{white}{\bfseries Proto} & \textcolor{white}{\bfseries Description} & \textcolor{white}{\bfseries Recovery} \\
\midrule

\posterinfo{
\textbf{MRP} uses ring topology with one blocked port. On link failure, the blocked port activates. \textbf{HSR/PRP} provide zero-loss redundancy by sending frames on two paths simultaneously.
}

\section{\textcolor{accent}{\faIcon{shield-alt}}\hspace{0.4em}Security Considerations}

\posterdanger{
\textbf{Common vulnerabilities:}
\begin{itemize}
    \item \textbf{No authentication:} Most fieldbus protocols accept any command
    \item \textbf{No encryption:} Traffic readable by anyone on the network
    \item \textbf{Flat networks:} No segmentation between zones
    \item \textbf{Legacy devices:} Cannot be patched or upgraded
    \item \textbf{Broadcast traffic:} Easy reconnaissance via passive sniffing
\end{itemize}
}

\postersuccess{
\textbf{Security measures:}
\begin{itemize}
    \item Segment networks by function and criticality (zones/conduits)
    \item Deploy industrial firewalls at zone boundaries
    \item Use managed switches with port security and VLANs
    \item Implement network monitoring and anomaly detection
    \item Disable unused ports and services on all devices
    \item Consider encrypted tunnels (VPN) for sensitive traffic
\end{itemize}
}

\subsection{\textcolor{accent}{\faIcon{tools}}\hspace{0.3em}Network Design Best Practices}

\begin{itemize}
    \item \textcolor{accent}{\faIcon{network-wired}}\hspace{0.2em}\textbf{Managed switches:} Enable port security, MAC filtering, and VLAN tagging to isolate traffic between zones
    \item \textcolor{accent}{\faIcon{shield-alt}}\hspace{0.2em}\textbf{Deep packet inspection:} Industrial-aware firewalls can parse OT protocols and enforce allowlists at the application layer
    \item \textcolor{accent}{\faIcon{search}}\hspace{0.2em}\textbf{Passive monitoring:} Use SPAN/mirror ports to monitor traffic without affecting real-time operations
    \item \textcolor{accent}{\faIcon{copy}}\hspace{0.2em}\textbf{Preserve redundancy:} Security implementations must not break MRP/HSR/PRP failover paths -- test thoroughly before deployment
\end{itemize}

\postertip{
\textbf{Key takeaway:} Topology matters -- star, ring, and bus have different reliability profiles. Serial fieldbus remains common in brownfield sites. Industrial Ethernet differs from IT Ethernet in determinism and redundancy requirements.
}
