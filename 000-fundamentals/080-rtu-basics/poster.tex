\section{\textcolor{accent}{\faIcon{broadcast-tower}}\hspace{0.4em}What is an RTU?}

A Remote Terminal Unit is a ruggedized, standalone device that interfaces with physical equipment at \textbf{remote locations} and communicates with a central SCADA system. RTUs are the eyes and ears of distributed infrastructure.

\posterinfo{
RTUs serve five key functions: \textbf{Acquire data} from sensors, \textbf{execute control} commands, \textbf{communicate} with SCADA masters, \textbf{store data} during outages, and \textbf{process locally} when communication is lost.
}

\postertip{
Modern devices blur the RTU/PLC distinction. Many RTUs include PLC-like programming; some PLCs support telemetry. ``RTU'' often indicates the device's \textbf{role} in SCADA architecture rather than strict hardware differences.
}

\section{\textcolor{accent}{\faIcon{microchip}}\hspace{0.4em}RTU Architecture}

\begin{itemize}
    \item \textcolor{accent}{\faIcon{microchip}}\hspace{0.2em}\textbf{CPU/Processor:} Executes control logic, manages comms
    \item \textcolor{accent}{\faIcon{memory}}\hspace{0.2em}\textbf{Memory (RAM/Flash):} Program, config, buffered data
    \item \textcolor{accent}{\faIcon{toggle-on}}\hspace{0.2em}\textbf{Digital I/O:} On/off status inputs and control outputs
    \item \textcolor{accent}{\faIcon{wave-square}}\hspace{0.2em}\textbf{Analog I/O:} 4-20mA, 0-10V measurement and control
    \item \textcolor{accent}{\faIcon{wifi}}\hspace{0.2em}\textbf{Comm Ports:} Serial, Ethernet, radio, cellular
    \item \textcolor{accent}{\faIcon{bolt}}\hspace{0.2em}\textbf{Power Supply:} AC/DC conversion with regulation
    \item \textcolor{accent}{\faIcon{battery-full}}\hspace{0.2em}\textbf{Battery Backup:} Maintains operation during outages
\end{itemize}

\subsection{\textcolor{accent}{\faIcon{exchange-alt}}\hspace{0.3em}Protocols}

\begin{itemize}
    \item \textcolor{accent}{\faIcon{network-wired}}\hspace{0.2em}\textbf{DNP3:} Distributed Network Protocol -- dominant in utilities
    \item \textcolor{accent}{\faIcon{exchange-alt}}\hspace{0.2em}\textbf{Modbus:} Simple, widely supported -- RTU and TCP variants
    \item \textcolor{accent}{\faIcon{globe}}\hspace{0.2em}\textbf{IEC 60870-5-101/104:} International telecontrol standard
    \item \textcolor{accent}{\faIcon{bolt}}\hspace{0.2em}\textbf{IEC 61850:} Modern substation automation
\end{itemize}

\section{\textcolor{accent}{\faIcon{shield-alt}}\hspace{0.4em}Security Considerations}

\posterdanger{
\textbf{Common vulnerabilities:}
\begin{itemize}
    \item \textcolor{danger}{\faIcon{unlock}}\hspace{0.2em}\textbf{No authentication:} Modbus, older DNP3 lack auth
    \item \textcolor{danger}{\faIcon{eye-slash}}\hspace{0.2em}\textbf{Cleartext comms:} Data transmitted without encryption
    \item \textbf{Physical access:} Remote sites may have weak security
    \item \textbf{Default credentials:} Factory passwords unchanged
    \item \textbf{Legacy firmware:} Difficult or impossible to update
\end{itemize}
}

\postersuccess{
\textbf{Security measures:}
\begin{itemize}
    \item \textbf{Protocol security:} DNP3 Secure Auth, IEC 62351
    \item \textbf{Encryption:} VPN tunnels, TLS for IP-based comms
    \item \textbf{Access control:} Strong passwords, disable unused ports
    \item \textbf{Physical security:} Locked enclosures, tamper detection
    \item \textbf{Monitoring:} Log communications, detect anomalies
    \item \textbf{Segmentation:} Separate RTU network from corporate IT
\end{itemize}
}

\posterwarning{
For new deployments, select RTUs supporting modern security (DNP3 SA, TLS, RBAC). For legacy devices, implement compensating controls: VPN tunnels, monitoring, and physical access controls.
}

\postertip{
\textbf{Key takeaway:} RTUs are the front-line interface between SCADA and the physical world at remote sites. Their geographic isolation creates unique security challenges -- physical access, communication link security, and autonomous operation during outages all require specific attention.
}
