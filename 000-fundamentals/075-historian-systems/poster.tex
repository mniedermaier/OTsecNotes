\section{\textcolor{accent}{\faIcon{database}}\hspace{0.4em}Overview}

Historians are specialized databases for collecting, storing, and retrieving \textbf{time-series data} from industrial processes. They capture measurements from sensors, PLCs, and SCADA systems at high speeds while maintaining long-term archives.

\posterinfo{
Historians bridge OT and IT networks (typically at Purdue Level 3), containing sensitive operational data and providing pathways attackers can exploit.
}

\section{\textcolor{accent}{\faIcon{chart-line}}\hspace{0.4em}Common Use Cases}

\begin{itemize}
    \item \textcolor{accent}{\faIcon{chart-bar}}\hspace{0.2em}\textbf{Process Optimization:} Analyze trends to improve efficiency
    \item \textcolor{accent}{\faIcon{search}}\hspace{0.2em}\textbf{Troubleshooting:} Review historical data during incidents
    \item \textcolor{accent}{\faIcon{clipboard-check}}\hspace{0.2em}\textbf{Compliance:} Maintain records for regulatory requirements
    \item \textcolor{accent}{\faIcon{tools}}\hspace{0.2em}\textbf{Predictive Maintenance:} Identify equipment degradation
    \item \textcolor{accent}{\faIcon{file-alt}}\hspace{0.2em}\textbf{Reporting:} Generate operational and management reports
\end{itemize}

\subsection{\textcolor{accent}{\faIcon{sitemap}}\hspace{0.3em}Deployment Patterns}

\begin{itemize}
    \item \textcolor{info}{\faIcon{server}}\hspace{0.2em}\textbf{Single Site:} One Historian, local access -- small facilities
    \item \textcolor{info}{\faIcon{network-wired}}\hspace{0.2em}\textbf{Distributed:} Site Historians with central aggregation -- multi-site
    \item \textcolor{info}{\faIcon{cloud}}\hspace{0.2em}\textbf{Cloud Hybrid:} Edge collection with cloud analytics -- IIoT
\end{itemize}

\subsection{\textcolor{accent}{\faIcon{stream}}\hspace{0.3em}Collection Methods}

\begin{itemize}
    \item \textbf{Polling:} Historian requests data at intervals
    \item \textbf{Exception-Based:} Sources send data on value change
    \item \textbf{Unsolicited:} Sources push data continuously
    \item \textbf{Store and Forward:} Buffer during network outages
\end{itemize}

\posterwarning{
\textbf{Compression settings affect data fidelity.} Aggressive compression may hide transient events or subtle anomalies important for security analysis and incident investigation.
}

\section{\textcolor{accent}{\faIcon{exclamation-triangle}}\hspace{0.4em}Security Threats}

\posterdanger{
\textbf{Why Historians are targets:}
\begin{itemize}
    \item \textcolor{danger}{\faIcon{eye}}\hspace{0.2em}Contain \textbf{sensitive process data} (operational intelligence)
    \item \textcolor{danger}{\faIcon{exchange-alt}}\hspace{0.2em}Sit at the \textbf{IT/OT boundary} -- pivot point for lateral movement
    \item \textcolor{danger}{\faIcon{network-wired}}\hspace{0.2em}Have \textbf{multiple network connections} to both zones
    \item \textcolor{danger}{\faIcon{user-shield}}\hspace{0.2em}Use \textbf{high-privilege accounts} for data collection
    \item \textcolor{danger}{\faIcon{server}}\hspace{0.2em}Often run on \textbf{legacy systems} with known vulnerabilities
\end{itemize}
}

\section{\textcolor{accent}{\faIcon{shield-alt}}\hspace{0.4em}Security Best Practices}

\postersuccess{
\textbf{Architecture:} Deploy Historians in DMZ between IT and OT. Use separate collector services in the OT zone with unidirectional or tightly controlled data flow to the Historian server.

\textbf{Controls:} Enforce strong authentication, enable TLS for data in transit, implement RBAC with least privilege, maintain patch schedule, log and alert on unusual queries, regular backups with integrity verification.
}

\textbf{Key principle:} Treat Historians as high-value targets. Their position at the IT/OT boundary and access to both zones makes them ideal pivot points for attackers moving laterally through the network.
