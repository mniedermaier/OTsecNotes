\section{\textcolor{accent}{\faIcon{sync-alt}}\hspace{0.4em}Lifecycle Overview}

The OT system lifecycle spans from procurement through decommissioning. Security must be integrated at every phase. Unlike IT systems with 3--5 year lifecycles, OT assets often operate for \textbf{15--30 years}.

\section{\textcolor{accent}{\faIcon{shopping-cart}}\hspace{0.4em}Procurement Phase}

\postersuccess{
\textbf{Security starts at procurement.} Specify requirements before purchase -- retrofitting security is expensive and often impossible with OT equipment.
}

\begin{itemize}
    \item \textcolor{accent}{\faIcon{lock}}\hspace{0.2em}\textbf{Authentication:} Strong auth, no hardcoded credentials
    \item \textcolor{accent}{\faIcon{key}}\hspace{0.2em}\textbf{Encryption:} TLS/secure protocols for communications
    \item \textcolor{accent}{\faIcon{clipboard-list}}\hspace{0.2em}\textbf{Logging:} Security event logging capabilities
    \item \textcolor{accent}{\faIcon{sync}}\hspace{0.2em}\textbf{Patching:} Vendor patch support commitment
    \item \textcolor{accent}{\faIcon{certificate}}\hspace{0.2em}\textbf{Certifications:} IEC 62443-4-1/4-2 compliance
\end{itemize}

\section{\textcolor{accent}{\faIcon{rocket}}\hspace{0.4em}Deployment Phase}

\subsection{\textcolor{accent}{\faIcon{list-ol}}\hspace{0.3em}Secure Configuration}

\begin{enumerate}
    \item \textcolor{success}{\faIcon{check}}\hspace{0.2em}Change default credentials immediately
    \item \textcolor{success}{\faIcon{check}}\hspace{0.2em}Disable unnecessary services and ports
    \item \textcolor{success}{\faIcon{check}}\hspace{0.2em}Configure network segmentation and firewalls
    \item \textcolor{success}{\faIcon{check}}\hspace{0.2em}Enable logging and connect to monitoring
    \item \textcolor{success}{\faIcon{check}}\hspace{0.2em}Document baseline and create backup
\end{enumerate}

\section{\textcolor{accent}{\faIcon{clock}}\hspace{0.4em}Operations Phase}

The longest phase -- often \textbf{15--30 years}. Security must be maintained as threats evolve.

\posterdanger{
\textbf{End-of-Support Systems:} When vendors stop patching, implement compensating controls: network isolation, application whitelisting, enhanced monitoring, and plan for replacement.
}

\subsection{\textcolor{accent}{\faIcon{user-lock}}\hspace{0.3em}Vendor \& Contractor Access}

\begin{itemize}
    \item Require documented authorization for each access session
    \item Escort or monitor vendor activities on-site
    \item Use time-limited, audited remote access only
    \item Verify all changes made during maintenance
    \item Scan vendor media before connecting to OT networks
\end{itemize}

\subsection{\textcolor{accent}{\faIcon{sync}}\hspace{0.3em}Patching Considerations}

\begin{itemize}
    \item \textbf{Test first:} Validate patches in lab environment
    \item \textbf{Schedule carefully:} Use planned maintenance windows
    \item \textbf{Have rollback plan:} Ability to restore if issues occur
    \item \textbf{Document everything:} Record patch levels and dates
    \item \textbf{Coordinate with vendor:} Ensure patches don't void support
\end{itemize}

\section{\textcolor{accent}{\faIcon{power-off}}\hspace{0.4em}Decommissioning Phase}

\posterdanger{
\textbf{Security doesn't end at shutdown.} Decommissioned equipment may contain credentials, network info, or process data exploitable if improperly disposed.
}

\begin{center}
\rowcolors{2}{lightgray}{white}
\begin{tabular}{p{2.2cm}p{4.8cm}}
\rowcolor{primary}
\textcolor{white}{\bfseries Data Type} & \textcolor{white}{\bfseries Sanitization Method} \\
\midrule
Config files & Factory reset, verify deletion \\
Credentials & Remove all accounts, reset defaults \\
Process data & Secure deletion or physical destruction \\
Network settings & Clear IPs, firewall rules \\
Logs & Export for retention, then delete \\
Firmware & Reinstall clean firmware \\
\end{tabular}
\end{center}

\subsection{\textcolor{accent}{\faIcon{clipboard-check}}\hspace{0.3em}Post-Decommission}

Remove from asset inventory, update network diagrams, revoke credentials, archive documentation, update firewall rules to remove references.

\postertip{
\textbf{Lifecycle thinking:} Security at every phase, not just operations. Procurement sets the foundation; decommissioning closes the loop. Plan for 15--30 year asset lifecycles from day one.
}
