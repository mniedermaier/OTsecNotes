\section{\textcolor{accent}{\faIcon{info-circle}}\hspace{0.4em}Overview}

Encryption protects data confidentiality and integrity, but OT environments present unique challenges: \textbf{real-time requirements, legacy systems, resource-constrained devices}, and deterministic communication needs. Many industrial protocols were designed without security when networks were isolated.

\subsection{\textcolor{accent}{\faIcon{lock}}\hspace{0.3em}MACsec (IEEE 802.1AE)}

\begin{itemize}
    \item \textcolor{accent}{\faIcon{lock}}\hspace{0.2em}Encrypts all traffic between switches at Layer 2
    \item \textcolor{warning}{\faIcon{clock}}\hspace{0.2em}Minimal latency impact (hardware-based)
    \item \textcolor{accent}{\faIcon{ethernet}}\hspace{0.2em}Requires MACsec-capable network equipment
    \item \textcolor{accent}{\faIcon{project-diagram}}\hspace{0.2em}Does not protect across Layer 3 boundaries
\end{itemize}

\postersuccess{
\textbf{MACsec} is well-suited for OT where low latency is critical and traffic stays within a Layer 2 domain. Industrial Ethernet switches increasingly support it.
}

\subsection{\textcolor{accent}{\faIcon{shield-alt}}\hspace{0.3em}IPsec (Layer 3)}

\begin{itemize}
    \item \textcolor{accent}{\faIcon{lock}}\hspace{0.2em}Transport mode: encrypt payload, preserve headers
    \item \textcolor{accent}{\faIcon{lock}}\hspace{0.2em}Tunnel mode: encrypt entire packet (VPN)
    \item \textcolor{accent}{\faIcon{desktop}}\hspace{0.2em}Supported by most modern operating systems
    \item \textcolor{accent}{\faIcon{bolt}}\hspace{0.2em}Hardware acceleration reduces overhead
\end{itemize}

\subsection{\textcolor{accent}{\faIcon{certificate}}\hspace{0.3em}TLS/DTLS (Layer 4)}

\begin{itemize}
    \item \textcolor{accent}{\faIcon{lock}}\hspace{0.2em}TLS 1.3 for TCP-based protocols
    \item \textcolor{accent}{\faIcon{lock}}\hspace{0.2em}DTLS for UDP-based (handles packet loss)
    \item \textcolor{accent}{\faIcon{exchange-alt}}\hspace{0.2em}Can wrap legacy protocols (e.g., Modbus/TCP + TLS)
    \item \textcolor{accent}{\faIcon{server}}\hspace{0.2em}Requires TLS-capable endpoints
\end{itemize}

\section{\textcolor{accent}{\faIcon{key}}\hspace{0.4em}Key Management}

\posterdanger{
\textbf{Poor key management undermines encryption.} Many OT breaches exploit weak or default keys rather than breaking encryption itself. Plan for full device lifecycle---15--25 year lifecycles exceed typical key validity.
}

\begin{itemize}
    \item \textcolor{accent}{\faIcon{lock}}\hspace{0.2em}\textbf{Centralized PKI} -- Certificate authority for OT
    \item \textcolor{accent}{\faIcon{lock}}\hspace{0.2em}\textbf{HSMs} -- Hardware secure key storage
    \item \textcolor{accent}{\faIcon{lock}}\hspace{0.2em}\textbf{Automated renewal} -- EST, CMP, or SCEP protocols
    \item \textcolor{accent}{\faIcon{tools}}\hspace{0.2em}\textbf{Offline procedures} -- Manual for air-gapped systems
    \item \textcolor{accent}{\faIcon{key}}\hspace{0.2em}\textbf{Key escrow} -- Backup keys for recovery
\end{itemize}

\postertip{
Choose encryption layer based on requirements: MACsec for low-latency L2, IPsec/TLS for flexibility, application-layer for native security. OPC UA and CIP Security are preferred for new deployments. Use TLS wrappers or VPNs for legacy protocols. Focus on zone boundaries and remote access first. Test performance impact before encrypting control traffic. Plan key management for the full device lifecycle.
}
