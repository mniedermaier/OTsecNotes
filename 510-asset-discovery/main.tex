\section{Introduction}

Asset discovery and inventory management is the foundation of any OT security program. You cannot protect assets you don't know exist, and you cannot assess risk without understanding what systems are in your environment.

\begin{infobox}
Industry reports from SANS ICS and Dragos indicate that organizations commonly underestimate their OT asset count by 30--50\%. Unknown assets represent unmanaged risk and potential attack vectors.
\end{infobox}

\subsection{Why Asset Inventory Matters}

\begin{itemize}
    \item \textbf{Vulnerability management:} Know what needs patching
    \item \textbf{Risk assessment:} Understand exposure and criticality
    \item \textbf{Incident response:} Identify affected systems quickly
    \item \textbf{Compliance:} Required by IEC 62443, NERC CIP, etc.
    \item \textbf{Change management:} Baseline for detecting unauthorized changes
\end{itemize}

\section{Discovery Methods}

\subsection{Passive Discovery}

\begin{successbox}
\textbf{Passive methods are preferred for OT environments:}
\begin{itemize}
    \item Network traffic analysis (SPAN/TAP)
    \item Protocol fingerprinting from observed communications
    \item DHCP and DNS log analysis
    \item Switch CAM table queries (low risk)
\end{itemize}
\end{successbox}

\subsection{Active Discovery}

\begin{warningbox}
\textbf{Active scanning risks in OT:}
\begin{itemize}
    \item May crash legacy PLCs and controllers
    \item Can trigger safety system responses
    \item May disrupt real-time communications
    \item Should only be performed during maintenance windows
\end{itemize}
\end{warningbox}

If active discovery is necessary:
\begin{itemize}
    \item Use OT-aware scanning tools with rate limiting
    \item Test in lab environment first
    \item Schedule during planned downtime
    \item Have rollback procedures ready
\end{itemize}

\subsection{Manual Methods}

\begin{itemize}
    \item \textbf{Physical walkdowns:} Visually inspect and document equipment
    \item \textbf{Documentation review:} Network diagrams, as-built drawings
    \item \textbf{Vendor records:} Maintenance contracts, support agreements
    \item \textbf{Interviews:} Operations and maintenance staff knowledge
\end{itemize}

\subsection{Security-Relevant Attributes}

\begin{itemize}
    \item \textbf{Network connectivity:} What can it communicate with?
    \item \textbf{Protocols used:} Modbus, OPC, Ethernet/IP, etc.
    \item \textbf{Authentication:} Does it support/require authentication?
    \item \textbf{Patch status:} Last update, available patches
    \item \textbf{End-of-life status:} Vendor support availability
    \item \textbf{Backup status:} Configuration backup available?
\end{itemize}

\section{Inventory Management}

\subsection{Initial Baseline}

\begin{enumerate}
    \item Deploy passive monitoring on key network segments
    \item Conduct physical walkdowns of critical areas
    \item Review existing documentation and diagrams
    \item Merge and deduplicate discovered assets
    \item Validate with operations and engineering staff
\end{enumerate}

\subsection{Ongoing Maintenance}

\begin{conceptbox}{Keeping Inventory Current}
\begin{itemize}
    \item \textbf{Continuous monitoring:} Detect new devices automatically
    \item \textbf{Change management integration:} Update on approved changes
    \item \textbf{Periodic reviews:} Quarterly validation with stakeholders
    \item \textbf{Reconciliation:} Compare discovered vs documented assets
\end{itemize}
\end{conceptbox}

\subsection{Common Challenges}

\begin{itemize}
    \item \textbf{Shadow OT:} Unauthorized devices connected by staff
    \item \textbf{Transient devices:} Vendor laptops, temporary equipment
    \item \textbf{Legacy systems:} Undocumented, forgotten equipment
    \item \textbf{Serial devices:} Not visible on IP networks
    \item \textbf{Air-gapped systems:} Require manual discovery
\end{itemize}

\section{Criticality Assessment}

\subsection{Classification Criteria}

Assess each asset's criticality based on:

\begin{itemize}
    \item \textbf{Safety impact:} Could failure cause injury?
    \item \textbf{Process impact:} Effect on production if unavailable
    \item \textbf{Recovery time:} How long to restore or replace?
    \item \textbf{Data sensitivity:} Confidential process information?
    \item \textbf{Connectivity:} Exposure to other networks
\end{itemize}

\subsection{Criticality Levels}

\begin{center}

\section{Integration}

\subsection{Connected Systems}

Asset inventory should integrate with:

\begin{itemize}
    \item \textbf{Vulnerability management:} Match assets to CVEs
    \item \textbf{SIEM/SOC:} Provide context for security alerts
    \item \textbf{CMDB:} Enterprise configuration management
    \item \textbf{Backup systems:} Ensure critical configs are backed up
    \item \textbf{Incident response:} Rapid asset lookup during incidents
\end{itemize}
