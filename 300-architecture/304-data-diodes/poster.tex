\section{\textcolor{accent}{\faIcon{long-arrow-alt-right}}\hspace{0.4em}Overview}

Data diodes (unidirectional security gateways) are \textbf{hardware-enforced one-way data transfer devices}. They physically prevent any data from flowing back to the source network -- the highest level of network isolation while still allowing data export.

\posterinfo{
\faIcon{lightbulb}\hspace{0.2em}Data diodes use fiber optic cables with a transmitter on one side and a receiver on the other -- with \textbf{no transmitter on the receiving side}. This makes reverse data flow physically impossible.
}

\section{\textcolor{accent}{\faIcon{cubes}}\hspace{0.4em}Components}

\begin{itemize}
    \item \faIcon{upload}\hspace{0.2em}\textbf{TX appliance:} Connects to source network, sends data
    \item \faIcon{dot-circle}\hspace{0.2em}\textbf{Optical fiber:} One-way light transmission medium
    \item \faIcon{download}\hspace{0.2em}\textbf{RX appliance:} Receives data, connects to destination
    \item \faIcon{ban}\hspace{0.2em}\textbf{No return fiber:} Physical absence of backward path
\end{itemize}

\posterwarning{
\faIcon{exclamation-triangle}\hspace{0.2em}TCP requires bidirectional communication (ACKs). Data diodes must use \textbf{protocol break/proxy} -- TX terminates TCP locally and retransmits via UDP. RX reconstructs the TCP connection.
}

\posterdanger{
\faIcon{ban}\hspace{0.2em}\textbf{When NOT to use:} Bidirectional communication required, remote access needed, cost prohibitive for risk level, or protocol support unavailable.
}

\section{\textcolor{accent}{\faIcon{project-diagram}}\hspace{0.4em}Deployment Pattern}

\postersuccess{
\textbf{Basic data export pattern:}
\begin{enumerate}
    \item \faIcon{database}\hspace{0.2em}OT historian collects process data
    \item \faIcon{upload}\hspace{0.2em}Data diode TX reads from OT historian
    \item \faIcon{long-arrow-alt-right}\hspace{0.2em}One-way transfer to RX appliance
    \item \faIcon{download}\hspace{0.2em}RX writes to replica historian in DMZ
    \item \faIcon{building}\hspace{0.2em}Enterprise apps read from replica
\end{enumerate}
}

\section{\textcolor{accent}{\faIcon{check-double}}\hspace{0.4em}Reliability Without ACKs}

Since there's no return path for acknowledgments:
\begin{itemize}
    \item \faIcon{code}\hspace{0.2em}\textbf{FEC:} Forward Error Correction codes
    \item \faIcon{clone}\hspace{0.2em}\textbf{Redundant TX:} Send data multiple times
    \item \faIcon{check-circle}\hspace{0.2em}\textbf{App-level verify:} Checks on receiving side
    \item \faIcon{hdd}\hspace{0.2em}\textbf{Buffering:} Store-and-forward on TX side
\end{itemize}

\section{\textcolor{accent}{\faIcon{shield-alt}}\hspace{0.4em}Best Practices}

\begin{enumerate}
    \item \faIcon{file-alt}\hspace{0.2em}Define data requirements and flow direction
    \item \faIcon{check-circle}\hspace{0.2em}Verify protocol support for your applications
    \item \faIcon{ban}\hspace{0.2em}Plan for no return path in application design
    \item \faIcon{tachometer-alt}\hspace{0.2em}Size for throughput and latency requirements
    \item \faIcon{flask}\hspace{0.2em}Test all data flows before production
    \item \faIcon{sitemap}\hspace{0.2em}Document architecture with flow diagrams
    \item \faIcon{tools}\hspace{0.2em}Plan maintenance for systems on both sides
\end{enumerate}

\subsection{\textcolor{accent}{\faIcon{exclamation-triangle}}\hspace{0.3em}Implementation Challenges}

\begin{itemize}
    \item \faIcon{sync}\hspace{0.2em}\textbf{Software updates:} Cannot push patches back to OT side
    \item \faIcon{heartbeat}\hspace{0.2em}\textbf{Health monitoring:} TX side status invisible from RX
    \item \faIcon{dollar-sign}\hspace{0.2em}\textbf{Cost:} Significantly more expensive than firewalls
    \item \faIcon{puzzle-piece}\hspace{0.2em}\textbf{Integration:} Requires custom app proxies for each protocol
    \item \faIcon{user-cog}\hspace{0.2em}\textbf{Skill gap:} Staff need training on unidirectional concepts
    \item \faIcon{clock}\hspace{0.2em}\textbf{Latency:} Protocol break adds processing delay to transfers
\end{itemize}

\postertip{
\faIcon{lightbulb}\hspace{0.2em}Data diodes provide \textbf{air-gap level security with data flow}. They complement firewalls -- use diodes at the most critical boundaries and firewalls where bidirectional communication is needed. Verify protocol support before procurement.
}
