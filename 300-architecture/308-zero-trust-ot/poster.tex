\section{\textcolor{accent}{\faIcon{lock}}\hspace{0.4em}Overview}

Zero Trust is based on \textbf{``never trust, always verify.''} Rather than assuming everything inside a network perimeter is safe, it requires continuous verification of every user, device, and connection. OT environments cannot simply adopt IT Zero Trust -- principles must be adapted for real-time requirements, legacy systems, and safety constraints.

\section{\textcolor{accent}{\faIcon{list-ol}}\hspace{0.4em}Core Principles}

\begin{enumerate}
    \item \faIcon{ban}\hspace{0.2em}\textbf{Never trust, always verify} -- No implicit trust
    \item \faIcon{bug}\hspace{0.2em}\textbf{Assume breach} -- Design for compromised systems
    \item \faIcon{fingerprint}\hspace{0.2em}\textbf{Verify explicitly} -- Authenticate every connection
    \item \faIcon{key}\hspace{0.2em}\textbf{Least privilege access} -- Minimal permissions
    \item \faIcon{th}\hspace{0.2em}\textbf{Micro-segment} -- Isolate assets individually
    \item \faIcon{eye}\hspace{0.2em}\textbf{Monitor everything} -- Log and analyze all activity
\end{enumerate}

}

\section{\textcolor{accent}{\faIcon{hard-hat}}\hspace{0.4em}Safety Considerations}

\begin{itemize}
    \item \faIcon{toggle-on}\hspace{0.2em}\textbf{Fail-open for safety} -- Safety comms continue if controls fail
    \item \faIcon{ban}\hspace{0.2em}\textbf{No inline blocking} on safety system paths
    \item \faIcon{eye}\hspace{0.2em}\textbf{Passive monitoring} rather than active blocking for SIS
    \item \faIcon{level-down-alt}\hspace{0.2em}\textbf{Graceful degradation} -- Reduced access, not halted ops
    \item \faIcon{hand-paper}\hspace{0.2em}\textbf{Manual override} -- Emergency bypass with logging
\end{itemize}

\postertip{
\faIcon{lightbulb}\hspace{0.2em}Most OT environments should target the \textbf{``Defined'' maturity level}. Full Zero Trust at Level 0/1 is often impractical. Focus verification at zone boundaries and human access points. Implement progressively from outside in.
}
