\section{\textcolor{accent}{\faIcon{shield-alt}}\hspace{0.4em}Overview}

Firewalls enforce network segmentation by controlling traffic between zones. In OT, firewalls must balance security with \textbf{operational constraints}: real-time performance, protocol support, and high availability demands differ significantly from IT deployments.

\subsection{\textcolor{accent}{\faIcon{search}}\hspace{0.3em}Deep Packet Inspection for OT}

DPI-capable firewalls inspect OT protocol content:

\begin{itemize}
    \item \faIcon{microchip}\hspace{0.2em}\textbf{Modbus:} Validate function codes, register addresses
    \item \faIcon{broadcast-tower}\hspace{0.2em}\textbf{DNP3:} Inspect object types, function codes
    \item \faIcon{plug}\hspace{0.2em}\textbf{OPC UA:} Validate service requests, node access
    \item \faIcon{industry}\hspace{0.2em}\textbf{EtherNet/IP:} Inspect CIP services and objects
    \item \faIcon{cog}\hspace{0.2em}\textbf{S7comm:} Monitor read/write operations to PLCs
\end{itemize}

\posterwarning{
\faIcon{exclamation-triangle}\hspace{0.2em}DPI for OT protocols requires \textbf{specific protocol support}. Verify coverage before deployment -- unsupported protocols are treated as generic TCP/UDP.
}

\subsection{\textcolor{accent}{\faIcon{copy}}\hspace{0.3em}High Availability}

\begin{itemize}
    \item \faIcon{pause}\hspace{0.2em}\textbf{Active/Passive:} Standby takes over on failure
    \item \faIcon{play}\hspace{0.2em}\textbf{Active/Active:} Load sharing with failover
    \item \faIcon{exclamation-circle}\hspace{0.2em}\textbf{Bypass mode:} Fail-open for critical paths (use cautiously)
\end{itemize}

\posterdanger{
\faIcon{exclamation-triangle}\hspace{0.2em}\textbf{Fail-open bypass} maintains availability but eliminates security. Use only where safety demands it, with compensating controls.
}

\section{\textcolor{accent}{\faIcon{cogs}}\hspace{0.4em}Configuration Best Practices}

\subsection{\textcolor{accent}{\faIcon{list-ol}}\hspace{0.3em}Rule Design Principles}

\begin{itemize}
    \item \faIcon{ban}\hspace{0.2em}\textbf{Default Deny} -- Block all not explicitly permitted
    \item \faIcon{key}\hspace{0.2em}\textbf{Least Privilege} -- Only required protocols/ports
    \item \faIcon{crosshairs}\hspace{0.2em}\textbf{Specificity} -- Use specific IPs, not broad subnets
    \item \faIcon{long-arrow-alt-right}\hspace{0.2em}\textbf{Direction Aware} -- Consider which side initiates
    \item \faIcon{file-alt}\hspace{0.2em}\textbf{Documented} -- Business justification per rule
\end{itemize}

\section{\textcolor{accent}{\faIcon{exclamation-circle}}\hspace{0.4em}Common Mistakes}

\begin{itemize}
    \item \textcolor{danger}{\faIcon{times}}\hspace{0.2em}\textbf{Any-to-any rules} -- defeats segmentation
    \item \textcolor{danger}{\faIcon{times}}\hspace{0.2em}\textbf{Disabled logging} -- no visibility
    \item \textcolor{danger}{\faIcon{times}}\hspace{0.2em}\textbf{No rule review} -- rules become permissive
    \item \textcolor{danger}{\faIcon{times}}\hspace{0.2em}\textbf{Temporary bypass} -- becomes permanent
    \item \textcolor{danger}{\faIcon{times}}\hspace{0.2em}\textbf{No outbound rules} -- misses exfiltration and C2
\end{itemize}

\section{\textcolor{accent}{\faIcon{eye}}\hspace{0.4em}Monitoring \& Maintenance}

\begin{itemize}
    \item \faIcon{clipboard-list}\hspace{0.2em}\textbf{Log} denied/allowed connections for detection
    \item \faIcon{history}\hspace{0.2em}\textbf{Audit} config changes for compliance
    \item \faIcon{heartbeat}\hspace{0.2em}\textbf{Monitor health:} CPU, memory, sessions
    \item \faIcon{calendar-alt}\hspace{0.2em}\textbf{Review rules} quarterly and after incidents
    \item \faIcon{database}\hspace{0.2em}\textbf{Integrate logs} with SIEM for visibility
\end{itemize}

\postertip{
\faIcon{lightbulb}\hspace{0.2em}Prioritize \textbf{low latency, high availability, and OT protocol support} over IT-centric features. Deploy redundant firewalls for production. Use DPI for protocol inspection, stateful for basic segmentation.
}
