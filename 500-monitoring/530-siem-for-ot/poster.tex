\section{\textcolor{accent}{\faIcon{info-circle}}\hspace{0.4em}Overview}

SIEM platforms aggregate, correlate, and analyze security data from across the enterprise. Extending SIEM to OT provides \textbf{unified visibility} but requires careful consideration of OT-specific data sources and operational constraints.

\posterinfo{
\textbf{SIEM benefits for OT:} Unified IT/OT visibility, cross-boundary attack correlation, centralized compliance logging, faster incident investigation, and long-term forensic storage.
}

\section{\textcolor{accent}{\faIcon{project-diagram}}\hspace{0.4em}Architecture}

\posterwarning{
\textbf{One-way data flow:} \textcolor{warning}{\faIcon{arrow-right}}\hspace{0.2em}Log data must flow from OT to IT/SIEM, never the reverse. Use data diodes or strict firewall rules to prevent SIEM access back into OT networks.
}

\subsection{\textcolor{accent}{\faIcon{link}}\hspace{0.3em}Correlation Rules}

\postersuccess{
\textbf{Multi-source correlation:}
\begin{itemize}
    \item \textcolor{success}{\faIcon{check-circle}}\hspace{0.2em}VPN login + FW allow + HMI access = normal session
    \item \textcolor{danger}{\faIcon{ban}}\hspace{0.2em}VPN login + FW deny + retries = potential attack
    \item \textcolor{warning}{\faIcon{user-secret}}\hspace{0.2em}Badge out + remote from inside = credential theft
    \item \textcolor{danger}{\faIcon{virus}}\hspace{0.2em}IT malware + OT traffic anomaly = lateral movement
\end{itemize}
}

\postertip{
Extend existing IT SIEM to OT using \textbf{one-way data flow} through DMZ log collectors. Ensure time synchronization across all sources. Build OT-specific detection rules---IT patterns don't translate directly. Correlate IT and OT events to detect cross-boundary attacks. Route OT alerts to personnel who understand industrial context.
}
